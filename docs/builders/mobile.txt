################################################################################
 Monstros
################################################################################

   Cada monstro no arquivo de monstros do VitaliaMUD, deve iniciar com um
sinal de sustenido (#) seguido de seu número, e terminar com a letra "E" ou
"S" em uma linha em branco, de acordo com o tipo de monstro.  Entre estes
marcadores, deve-se inserir os elementos que dão as características ao
monstro, neste formato:

------------------------
 #<vnum>
 <ids>~
 <nome-curto>~
 <nome-longo>
 ~
 <descrição>
 ~
 <flags> <aff> <alinh> <tipo>
 {informações específicas}
 <tipo>
------------------------

   VNUM
   O número do monstro.  O número deve estar dentro do intervalo de números da
área em construção.  Os monstros devem ser dispostos no arquivo em ordem
numérica crescente, do menor para o maior.

   IDS
   Os nomes que identificam o monstro (que são usados em KILL, LOOK, KICK,
TRACK, etc).  Coloque aqui nomes que podem ser identificados tanto se o
jogador ler o NOME-CURTO, quanto o NOME-LONGO.
   Em versões anteriores do VitaliaMUD, costumava-se duplicar os nomes que
possuiam acentos, deixando um dos nomes sem acentos, para que os jogadores
que não tivessem a capacidade de digitá-los pudessem ainda assim usar os
comandos.  Contudo, agora não há mais tal necessidade, pois o MUD já possui
formas de reconhecer e comparar palavras acentuadas e não-acentuadas.
   Você não deve incluir artigos, pronomes e outros nome menos importantes
nesse campo.  Seria muito estranho e muito perigoso um id "o dragão".
Você poderia usar o comando "kill o" para atacar tal mob, por exemplo.

   NOME-CURTO
   Este é o nome curto do monstro.  Este nome não deve começar com letra
maiúscula, exceto se for o nome próprio do monstro (ex: "Júpiter").  De uma
forma geral, esse nome deve vir acompanhado de um artigo (o, a, um, uma).
Esse nome será usado nas mensagens de luta, por exemplo:
"Você tenta retalhar *um lenhador* mas erra por quilômetros."

   NOME-LONGO
   Este é o texto que irá aparecer quando um jogador entrar ou olhar a sala
onde o monstro estiver.  Deve ser uma mensagem dizendo que o monstro está no
lugar e o que ele se encontra fazendo.  Deve-se deixar claro nesta frase uma
ou mais das palavras que estão no campo IDS, para que o jogador saiba o nome
do monstro para usar com os comandos KILL, LOOK, etc. Por exemplo:
"Um lenhador está aqui, descansando de seu árduo trabalho."

   DESCRIÇÃO
   Uma descrição de algumas linhas da aparência do monstro.  Será mostrada
quando o jogador digitar LOOK seguido de um dos IDs do monstro.

   FLAGS
   Este campo contém as flags que definem várias características do monstro.
Este campo pode ser escrito de duas formas: usando letras, onde cada letra
indica uma das flags do monstro; e usando números, que indica a soma do valor
de cada uma das flags.  O primeiro método, usando letras, é preferível, pois
permite uma melhor leitura.  As diversas flags de monstros são descritas mais
adiante, na seção "Flags".

   AFF
   Este campo contém as flags de afetamento do monstro.  Como no campo FLAGS,
pode-se escrever este campo de duas formas, usando letras ou números, porém é
preferível o uso de letras.  As diversas flags de afetamento são descritas
no arquivo "aff-flags.txt".

   ALINH
   O alinhamento do monstro.  Valores iguais ou inferiores a -350 indicam que
o monstro é mau (possui aura avermelhada), valores entre -350 e 350
indicam que o monstro é neutro, e valores iguais ou superiores a 350 indicam
que o monstro é bom (possui aura azulada).  Ao matar monstros de determinado
alinhamento, o alinhamento do jogador tende a se tornar o inverso do
alinhamento do monstro.

   TIPO
   Define o tipo do monstro.  Atualmente existem 2 tipos de monstros: os
monstros tipo "S" (de Simple) e os monstros tipo "E" (de Enhanced).  Ao
definir o tipo de monstro, o campo INFORMAÇÕES ESPECÍFICAS aceitará
determinados campos, de acordo com o tipo.  As informações que cada um dos
tipos de monstro aceitam são descritas mais adiante, nas seções "Informações
Específicas de monstros tipo 'S'" e "Informações específicas de monstros tipo
'E'".
   Ao terminar de descrever as Informações Específicas do monstro, a mesma
letra que foi usada para definir o tipo do monstro deverá ser repetida, em uma
linha em branco, indicando que alí termina o monstro, e começa o seguinte.


================================================================================
 Flags
--------------------------------------------------------------------------------

   a. NOSPEC (1)
   Desativa funções especiais.  Essa flag não deve ser usada nos arquivos de
monstros.

   b. SENTINEL (2)
   O monstro irá ficar parado na sala onde ele estiver, a menos que seja
controlado por um Deus, ou por magia.  Sem esta flag, o monstro anda através
das salas, automaticamente.

   c. SCAVENGER (4)
   O monstro irá tomar para sí os objetos que forem deixados ao chão da sala
que ele se encontrar.

   d. NOPOISON (8)
   Não pode ser envenenado (magia poison).

   e. AWARE (16)
   A habilidade backstab não terá sucesso quando usada contra o monstro.

   f. AGGRESSIVE (32)
   O monstro irá atacar qualquer personagem (que não seja outro monstro) que
estiver na mesma sala.

   g. STAY_ZONE (64)
   Para monstros sem a flag SENTINAL, a presença desta flag não permitirá que
o monstro saia da área da qual faz parte.

   h. WIMPY (128)
   Se durante a luta, a quantidade de HP (Hit Points) do monstro cair para
menos de 20% do total, ele irá fugir da luta automaticamente.  Se esta flag
for usada em conjunto com a flag AGGRESSIVE, o monstro irá atacar apenas os
personagens que estiverem dormindo.

   i. AGGR_EVIL (256)
   j. AGGR_GOOD (512)
   k. AGGR_NEUTRAL (1024)
   Como a flag AGGRESSIVE, porém, o monstro só atacará personagens maus (flag
AGGR_EVIL), personagens bons (flag AGGR_GOOD) ou neutros (flag AGGR_NEUTRAL).
Mais de uma dessas flags podem ser usadas ao mesmo tempo.

   l. MEMORY (2048)
   O monstro irá se lembrar dos jogadores que o atacarem, e irão atacar sempre
que os verem.

   m. HELPER (4096)
   Ajuda outros monstros de mesmo alinhamento que estiverem lutando na sala.

   n. NOCHARM (8192)
   Não pode ser encantado (magia charm person).

   o. NOSUMMON (16384)
   Não pode ser convocado (magia summon).

   p. NOSLEEP (32768)
   Não pode ser forçado a dormir (magia sleep).

   q. NOBASH (65536)
   Não pode ser derrubado (habilidade bash).

   r. NOBLIND (131072)
   Não pode ser cegado (magia blind).

   s. MOUNTABLE (262144)
   Pode ser montado por outros personagens.

   t. CAN-FLY (524288)
   Monstros alados, ou com capacidade para voar.

   u. NOKILL (2097152)
   Não pode ser atacado.

   v. ISNPC (4194304)
   Esta flag é automaticamente definida para todos os monstros (NPCs).
   NÃO DEVE SER USADA em arquivos de mobs.

   w. NOTDEADYET (8388608)
   Esta flag é para uso interno durante a extração de monstros.
   NÃO DEVE SER USADA em arquivos de mobs.

   x. BRAVE (16777216)
   Monstro lutará até a morte, não fugirá mesmo com HP baixo.


================================================================================
 Informações Específicas de monstros tipo 'S'
--------------------------------------------------------------------------------

------------------------
 <nível> <thac0> <ac> <maxhp> <bhd>
 <ouro> <exp>
 <pos-carr> <pos-padr> <sexo>
------------------------

   NÍVEL
   O nível do monstro.  Deve ser compatível com o nível que se espera que um
jogador tenha para poder matá-lo com certa dificuldade.  O comando CONSIDER
utiliza-se deste campo para calcular a dificuldade para se lutar contra o
monstro.

   THAC0 ("To Hit Armor Class Zero")
   Equivale a média do número de erros, em 20 rounds de luta.  Dessa forma, o
número pode variar entre 0 e 20.  A inteligência e a sabedoria diminuem o
número de erros.  Quanto menor esse número, menor é a possibilidade do monstro
errar seus golpes.

-=> Se você tiver dificuldades de imaginar isso, é só lembrar que o THAC0 é
uma espécie de Hitroll ao contrário...  Já que quanto maior o hitroll,
melhor ele é (mais acertos de golpes), quanto menor THAC0, melhor (menos
erros de golpes) ele será.

-=> O THAC0 pode ser negativo (o Hitroll ainda maior), só que não há real
efeito prático nisso.

   AC ("Armor Class")
   É a classe da armadura do monstro quando estiver sem nenhum equipamento.
Este é o número *real* da classe da armadura, não confunda com o número que é
mostrado nos comandos SCORE e STAT.  O AC nesses comandos é representado como
uma fração de 10, aqui ele deve ser representado na forma inteira.  Exemplos:
      fração de dez | inteiro      fração de dez | inteiro
      --------------+--------      --------------+--------
        100/10      |   10           -10/10      |   -1
         40/10      |    4          -120/10      |  -12
          0/10      |    0          -190/10      |  -19

-=> PS:
Para ser mais fácil, uma curta explicação:
Se você quer um ac -80 (no jogo) pro mob, deve escrever -8.  Um mob com -80
de ac aqui terá lá no mud -800 de AC...  Sendo assim, nunca haverá um mob
com AC -79, -44...  Apenas se "SETarem" isso nele.

   MAXHP
   O valor de saúde máximo que o monstro deverá ter.  Esse valor deve ser
fornecido no formato de dados: <x>d<y>+<z>, onde X é o número de lados do
dado, Y é o tamanho do dado, e Z é o bônus, ou parte fixa do valor.  Exemplos:
   4d6+800  ->  valores entre 801 e 824
   8d12+1800  ->  valores entre 1812 e 1896
   Para mais informações sobre como usar este campo, veja mais adiante a seção
"Entendendo os Dados".

   BHD  ("Bare Hand Damage")
   É o dano sem armas que o monstro aplicará sobre seus oponentes.  Esse valor
deve ser fornecido no formato de dados: <x>d<y>+<z>, onde X é o número de
lados do dado, Y é o tamanho do dado, e Z é o bônus, ou parte fixa do valor.
Exemplos:
   4d6+2  ->  valores entre 8 e 26
   3d5+1  ->  valores entre 6 e 16
   Ao empunhar uma arma, o dano passa a ser de acordo com a arma que for
empunhada, mas mantendo o bônus.  Se, por exemplo, o monstro tiver um BHD de
4d6+2, e empunhar uma arma de dano 5d8, o seu dano total será 5d8+2 a cada
round.  Para mais informações sobre como usar este campo, veja mais adiante a
seção "Entendendo os Dados".
   A componente Z desse dado é o DAMROLL do monstro.

-=> Esse dica é para qualquer lugar que use o formato XdY+Z: Para ter uma
base sobre quanto o dado dá, você deve usar o dano médio.  E para uma
consulta rápida, tenha uma listinha dos dados e respectivos danos.  Evita
muito trabalho isso.

   OURO
   A quantidade de moedas de ouro que o monstro irá carregar.

   EXP
   A quantidade de experiência que o monstro irá possuir.  Note que quando o
monstro for morto, apenas um terço de sua experiência é passada para o
personagem que o matou.

-=> Melhor XP a menos que a mais... Cuidado com exageros.

   POS-CARR, POS-PADR
   Indicam a posição do monstro (sentado, em pé, etc), e são,
respectivamente, a posição em que o monstro será carregado e a sua posição
padrão.  A posição de carregamento é a que o monstro estará quando for
carregado no jogo.  A posição padrão é a posição que indica quando o MUD
irá usar a mensagem descrita no campo NOME-LONGO.  Deve ser um número entre
estes:
	3 -> Meditando
	4 -> Dormindo
	5 -> Descansando
	6 -> Sentado
	8 -> Em pé

   SEXO
   Indica o sexo do monstro.  Influencia diretamente os artigos que serão
usados para o MUD mostrar mensagens se referindo ao monstro.  Deve ser um
número entre estes:
	0 -> Neutro (em desuso, serve apenas para a língua inglesa)
	1 -> Masculino (ex: O guardião está sentado)
	2 -> Feminino (ex: A guardiã está sentada)

================================================================================
 Informações Específicas de monstros tipo 'E'
--------------------------------------------------------------------------------

------------------------
 <nivel> <thac0> <ac> <maxhp> <bhd>
 <ouro> <exp>
 <pos-carr> <pos-padr> <sexo>
 {atributos adicionais}
------------------------

   NIVEL, THAC0, AC, MAXHP, BHD, OURO, EXP, POS-CARR, POS-PADR, SEXO
   Os mesmos dos monstros tipo 'S'.  Ver explicação a eles acima.

   ATRIBUTOS ADICIONAIS
   São os atributos escritos seletivamente, cada um em uma linha, que indicam
outras características dos monstros.  Não há a necessidade de definir todos os
campos, apenas os que se desejar definir.  São os que seguem:

------------------------
 BareHandAttack: <valor>
 Str: <valor>
 StrAdd: <valor>
 Int: <valor>
 Wis: <valor>
 Dex: <valor>
 Con: <valor>
 Cha: <valor>
------------------------

   BAREHANDATTACK
   É usado para definir que tipo de mensagem de ataque o MUD irá mostrar
enquanto o monstro estiver atacando sem segurar nenhuma arma.  Deve ser um
número de 0 até 14, conforme descrito no arquivo "attack.txt".

   STR, INT, WIS, DEX, CON, CHA
   São, respectivamente, a força, inteligência, sabedoria, destreza,
constrituição e carisma do monstro.  Os valores para estes campos devem
estar entre 3 e 25.

   STRADD
   Se o campo STR for 18, este valor poderá ser usado, indicando a força
adicional.  O valor deve estar entre 0 e 100.

-=>  Se você não entendeu bem a diferença entre mobs S e E, deve se lembrar que
normalmente vê mobs com essas características:

Str: [11/0]  Int: [11]  Wis: [11]  Dex: [11]  Con: [11]  Cha: [11]

   São mobs comuns, nem sempre os mais fortes.  Já um mob E pode ter essas
características mudadas, às vezes para mais que isso, identificando os mais
poderosos:

Str: [25/0]  Int: [25]  Wis: [25]  Dex: [25]  Con: [25]  Cha: [25]
(esse é o Júpiter)

   Não se esqueça que nada lhe impede de fazer um mob fraco com
características como as de um jogador, por exemplo:
Str: [14/0]  Int: [17]  Wis: [15]  Dex: [16]  Con: [10]  Cha: [6]


================================================================================
 Entendendo os Dados
--------------------------------------------------------------------------------

   Os dados são conhecidos em 2 formas diferentes: <x>d<y> e <x>d<y>+<z>.  Os
valores significam:

   X -> É o número do primeiro lado do dado.
   Y -> É o tamanho (número de lados) do dado.
   Z -> É o bonus, que é adicionado ao número final lançado pelo dado.

   Como exemplo, um dado simples 1d6 equivaleria a um dado comum de seis
lados, cujo o primeiro lado é 1, e cada lado é 1 número maior que o anterior.
Um dado 4d6 equivale a um dado de 6 lados, e cada lado e 4 números maior que o
anterior.

   Alguns exemplos:
	1d6 = { 1, 2, 3, 4, 5, 6 }		média =  3,5
	4d6 = { 4, 8, 12, 16, 20, 24 }		média = 14,0
	3d8 = { 3, 6, 9, 12, 15, 18, 21, 24 }	média = 13,5

   O menor número que um dado simples pode resultar é o mesmo valor de X, e o
maior é o resultado da multiplicação de X e Y.  A média pode ser obtida
através do seguite cálculo:

          x ( y + 1 )
média = -------------- = x * (y + 1) / 2
              2

   Quando o dado possui um bonus, esse último número é adicionado ao valor do
lançamento do dado.  Por exemplo, um dado 4d6+5 será igual a um dos números 4,
8, 12, 16, 20 ou 24 (os 6 lados do dado) somado de 5.  Por exemplo, um monstro
com um MAXHP de 10d6+800 poderá ter um dos seguintes valores quando
carregado:
	10d6+800 = { 810, 820, 830, 840, 850, 860 }
